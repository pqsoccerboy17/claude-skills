# Client Portal - Spec Process

Read the factory-pm SKILL.md. Following the spec-before-code workflow.

## Phase 1: Context Gathered (silent)
Read CLAUDE.md, package.json, directory structure.

## Phase 2: Clarifying Questions

Before I write a spec, I have a few questions:

1. **Scope**: Should clients be able to leave comments or feedback on deliverables, or is this view-only?
2. **Users**: How many clients do you anticipate? Will they each have individual logins, or will some clients have multiple team members?
3. **Integration**: Should invoices sync with your existing invoicing tool (e.g., Stripe, QuickBooks), or will this be standalone?
4. **Constraints**: Do you have a preferred tech stack, or should I recommend one based on the project?
5. **Priority**: What's the single most important thing a client should be able to do on day one?

*Waiting for answers before writing the spec...*

## Phase 3: Spec (written after answers)

Would write to specs/client-portal.md with:
- Summary
- P0/P1/P2 requirements
- Non-goals
- Technical approach
- Implementation steps

## Phase 4: Approval Gate

"Spec written to specs/client-portal.md. Review it above. When you're ready, reply Approved to start implementation, or tell me what to change."

No code will be written until explicit approval.
